# Introduction

To construct a pipeline to use and evaluate the cuVGL package. This include two parts:
1. Map construction with the [Isaac Mapping ROS package](https://nvidia-isaac-ros.github.io/v/release-3.2/repositories_and_packages/isaac_ros_mapping_and_localization/isaac_mapping_ros/index.html). We currently use a end-to-end map creation script provided by this package, the relevant usages can be found in [Tutorial: Mapping and Localization with Isaac Perceptor](https://nvidia-isaac-ros.github.io/v/release-3.2/reference_workflows/isaac_perceptor/tutorial_mapping_and_localization.html#creating-maps).
2. Global localization using the cuVGL package. This package depends on the map generated by Isaac Mapping ROS. We have wrote a ROS2 launch file to start it together with cuVSLAM.

This is a workspace based on the [Isaac ROS official Doc](https://nvidia-isaac-ros.github.io/v/release-4.1/getting_started/index.html#create-a-workspace). All the Isaac Ros functionalities are run in a [containerized environment](https://nvidia-isaac-ros.github.io/v/release-3.2/concepts/docker_devenv/index.html#development-environment).

# System Requirement

- Hardware: Jetson Orin
- Ros2 version: Humble
- Isaac Ros version: 3.2

# Dependencies

## ZED ROS 2 Wrapper

In case you need to use the [frame name remapping script](src/mapping/scripts/remap_frames.py) to fix the base frame name, you will have to install this package:

```bash
sudo apt install ros-<ros2-distro>-zed-msgs
```

# Quick Start

## On local machine

Set Up Development Environment

```bash
just set_up_development_environment
```

Enter the dev container

```bash
just docker
```

## Map Creation

Create the maps (vslam map and vgl map)

```bash
cd src/mapping
make create_map_offline
```

## Test cuVGL

Build/install launch files

```bash
cd src/isaac_localization_launch
make build
source install/setup.bash
```

Remove build artifacts

```bash
make clean
```

Run cuVGL

```bash
make run
# then open another terminal
make replay
```

# Notes

- cuVSLAM should receive images of type mono8, rgb8, while bgra8 is not accepted.
- cuVGL can read mono8, rgb8, and bgra8.

If you need to use ZED inside Isaac ROS Docker container, see [StereoLabs docs](https://www.stereolabs.com/docs/isaac-ros/setting_up_isaac_ros#configure-nvidia-isaac-ros-using-docker-with-zed-cameras).
